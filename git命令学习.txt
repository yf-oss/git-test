Git 工作区、暂存区和版本库概念：
工作区：就是你在电脑里能看到的目录。
暂存区：英文叫 stage 或 index。一般存放在 .git 目录下的 index 文件（.git/index）中，
所以我们把暂存区有时也叫作索引（index）。
版本库：工作区有一个隐藏目录 .git，这个不算工作区，而是 Git 的版本库。

一、一般工作流程如下：
1、克隆 Git 资源作为工作目录。
2、在克隆的资源上添加或修改文件。
3、如果其他人修改了，你可以更新资源。
4、在提交前查看修改。
5、提交修改。
6、在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

二、终端git命令：
1、克隆项目：使用git clone 命令将远程仓库中的代码克隆到本地。
git clone git仓库地址
如果指定克隆到哪个目录使用 git clone git仓库地址 项目名称

2、拉取代码：
下载远程代码并合并：git pull

3、提交代码：使用git add 将本地修改过的文件添加到git缓存git中，使用git commit将缓存中的文件
提交到本地仓库，最后使用git push命令将本地的仓库推送到远程仓库

添加文件到暂存区:git add .
将暂存区内容添加到本地仓库中:git commit -m "提交描述"

4、上传代码：使用git push 命令将本地代码上传到远程仓库。
上传远程代码并合并：git push



三、常用命令：

初始化一个git仓库：git init

查看当前git配置：git config -list

查看仓库当前的状态，显示有变更的文件：git status
